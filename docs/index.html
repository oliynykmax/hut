🏠 hut
Features GitHub

A fast build system for C/C++

Fast, zero‑config build system and package manager. No CMake, no Makefiles — just write C and run.

$ curl -fsSL https://raw.githubusercontent.com/oliynykmax/hut/main/install.sh | bash
terminal — hut workflow 
# Create a new project
$ hut init hello && cd hello

# Build and run — zero config
$ hut run
Compiling src/main.c
Finished release in 0.03s
Hello, world!

# Add a dependency
$ hut a raylib
Resolved raylib ^5.5.0
Installed 1 package in 0.8s
⚡

Zero config

Drop a main.c and run. No CMakeLists.txt, no Makefiles, no build scripts. hut figures it out.

📦

Package manager

Add dependencies with hut add. Resolves versions, fetches source, links automatically. One command, done.

🔥

JIT compilation

Run C files instantly with hut run --jit. No binaries written — compiles in‑memory via libtcc.

🎯

Compiler choice

GCC, Clang — your call. hut auto‑detects what's available and lets you pick. Or set it once and forget.

🧩

Header‑only libs

Pulling a header‑only library? hut resolves includes, generates a unified header dir, and wires up -I flags.

🔄

Self‑updating

hut upgrade pulls the latest source, rebuilds, and replaces the binary — all in one command.

🏗️

Workspaces

Monorepo support. Run commands across packages, manage shared deps, link local crates together.

🧹

Clean & fast

Rust under the hood. Parallel builds, incremental linking, hut clean to free space. No bloat.

⌨️

Short aliases

hut b, hut i, hut a, hut rm, hut up, hut t — short and muscle‑memory friendly.
